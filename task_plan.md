# 任务计划：文档章节结构改造

## 目标
将文档生成逻辑从"每个手风琴独立章节"改为"第一个手风琴生成'1 概述'，其余手风琴生成'2 分析过程及结果'的子章节"，并为每个章节增加用户可编辑的图片组标题。

## 当前状态
- **阶段**: 准备执行
- **开始时间**: 2026-04-08
- **预计改动文件**: 3个（index.html, report_tool.py, section_builders.py）

---

## 改造阶段

### Phase 1: 前端改动（index.html）
**状态**: pending
**预估时间**: 15分钟

**改动内容**:
1. 在每个手风琴面板增加"图片组标题"输入框
2. 修改 `imageConfig` 数据结构，增加 `groupTitle` 字段
3. 修改表单提交逻辑，包含 `groupTitle` 字段

**验证方式**:
- [ ] 前端显示新增的输入框
- [ ] `imageConfig` 包含 `groupTitle` 字段
- [ ] 提交表单时 `groupTitle` 正确传递到后端

---

### Phase 2: 章节构建器改动（section_builders.py）
**状态**: pending
**预估时间**: 10分钟

**改动内容**:
1. 修改 `build_complete_section()` 函数签名，增加 `figure_caption` 参数
2. 新增 `build_figure_caption()` 函数，构建居中图题段落
3. 在图片槽位后、VLM分析前插入图题

**验证方式**:
- [ ] 函数签名正确更新
- [ ] 图题正确插入到文档中
- [ ] 图题格式符合要求（居中、宋体 10.5pt）

---

### Phase 3: 后端主逻辑改动（report_tool.py）
**状态**: pending
**预估时间**: 20分钟

**改动内容**:
1. 第一个手风琴生成"1 概述"章节，图题为"图1 [groupTitle]"
2. 先插入"2 分析过程及结果"大标题
3. 其余手风琴生成子章节 2.1、2.2...，图题为"图2.1 [groupTitle]"
4. 传递 `figure_caption` 参数到 `build_complete_section()`

**验证方式**:
- [ ] 第一个手风琴生成"1 概述"
- [ ] 其余手风琴生成"2.x"子章节
- [ ] 图题编号正确（图1、图2.1、图2.2...）
- [ ] 文档结构与预期一致

---

### Phase 4: 集成测试
**状态**: pending
**预估时间**: 10分钟

**测试场景**:

3. 每个章节上传2-3张图片 → 验证图片编号 (a)、(b)、(c)
4. 填写图片组标题 → 验证图题显示正确

**验证方式**:
- [ ] 所有测试场景通过
- [ ] 生成的文档格式符合要求

## 关键决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 图片组标题输入框位置 | 每个手风琴面板内，上传区域之前 | 与槽位label同层级，语义清晰 |
| 图题编号规则 | 第一个为"图1"，其余为"图2.1"、"图2.2"... | 符合文档层级结构 |
| 图题默认值 | 无默认值，用户必须填写 | 避免生成空图题 |
